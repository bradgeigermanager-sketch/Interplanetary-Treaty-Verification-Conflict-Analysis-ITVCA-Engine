To effectively model human behavior in multi-planetary legal scenarios, the ITVCA engine must transition from a deterministic logic system to a **Probabilistic Deontic Logic (PDL)**. Criminality is fundamentally tied to *Mens Rea* (guilty mind), which is inherently an epistemic state—meaning it depends on what an agent knew or intended at the time of action.

### 1. Expanded Logical Syntax: The Epistemic/Intentional Layer

We append a **Cognitive State Vector** to our 7-tuple. The rule now expands to a 9-tuple: `rule(ID, Actor, Op, Action, Condition, Constraint, Liability, Epistemic_State, Intent_State)`.

* **Epistemic State ($K$):** Represents knowledge or lack thereof.
* $K(\phi)$: The agent knows proposition $\phi$.
* $\neg K(\phi)$: The agent is ignorant of proposition $\phi$ (e.g., unaware of a planetary law).


* **Intent State ($I$):** Represents the actor’s psychological drive.
* $I_{mal}(\phi)$: Malicious intent toward $\phi$.
* $I_{ben}(\phi)$: Benign/Accidental intent toward $\phi$.
* $I_{corrupt}(\phi)$: Intent to subvert the legal system for gain.

% Rule for "Ignorance of Law" (Mitigation)

    rule(ID, Actor, obligatory, Action, _, _, criminal, neg K(law), I_ben(Action)),
    apply_penalty(reduction).

### 3. Conflict Resolution with Corrupt Intent

When an agent acts with $I_{corrupt}$, the meta-norms of the system shift. The system uses a **Corrupt Intent Override**:

$$I_{corrupt} \implies \text{Nullify}(Treaty\_Immunity)$$

If the system detects that an agent utilized a treaty or diplomatic status specifically to facilitate a crime, the $I_{corrupt}$ flag triggers the engine to bypass standard treaty protections. This prevents criminals from hiding behind the jurisdictional complexities we modeled earlier.

### 4. Updated Procedure: The "Mens Rea" Audit

When analyzing a crime in progress or post-facto, use this procedure to integrate the epistemic/intent layer:

1. **Epistemic Mapping:** Establish the "Information Set" available to the actor. Was the planetary law published? Was the treaty text accessible to the individual or corporation?
2. **Intent Classification:** Categorize the action as $I_{mal}$, $I_{ben}$, or $I_{corrupt}$ based on the sequence of events (e.g., did they attempt to hide evidence?).
3. **Liability Weighting:**
* If $K(\phi) \land I_{mal}(\phi) \to$ Full Liability.
* If $\neg K(\phi) \land I_{ben}(\phi) \to$ Mitigation or Warning.
* If $I_{corrupt} \to$ Invoke "Good Faith" exception clause (denial of immunity).


4. **Conflict Re-Evaluation:** If a treaty requires immunity for a diplomat, but $I_{corrupt}$ is proven, the engine generates a *Jurisdictional Override Request*.
